\chapter{Dialogs}

Standard dialogs are one of the highest-value parts of the binding because they unlock native-feeling file selection, confirmation flows, color picking, font selection, text input, and progress reporting.

\section{Convenience Helpers}

Helpers create a modal dialog, execute it, return the chosen value, and release the native dialog.

\begin{lstlisting}[style=crystal]
if color = Qt6::ColorDialog.get_color(
  main,
  current_color: Qt6::Color.new(52, 112, 176),
  title: "Pick Accent Color",
  show_alpha_channel: true
)
  status_bar.show_message("Selected #{color.red},#{color.green},#{color.blue}")
end
\end{lstlisting}

\section{Instance Configuration}

Use explicit dialog instances when you need to configure modes before execution or inspect state after acceptance.

\begin{lstlisting}[style=crystal]
dialog = Qt6::FileDialog.new(main, Dir.current, "Maps (*.map *.json);;All Files (*)")
dialog.window_title = "Open Map"
dialog.accept_mode = Qt6::FileDialogAcceptMode::Open
dialog.file_mode = Qt6::FileDialogFileMode::ExistingFile

if dialog.exec == Qt6::DialogCode::Accepted
  status_bar.show_message(dialog.selected_file)
end
\end{lstlisting}

\section{Current Coverage}

The dialog surface includes \api{MessageBox}, \api{FileDialog}, \api{ColorDialog}, \api{FontDialog}, \api{InputDialog}, \api{ProgressDialog}, \api{SplashScreen}, \api{DialogButtonBox}, and base \api{Dialog} accepted/rejected callbacks.
