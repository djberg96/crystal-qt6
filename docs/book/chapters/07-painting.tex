\chapter{Painting And Vector Geometry}

The rendering layer centers on \api{QPainter}, \api{QPainterPath}, \api{QTransform}, \api{QPen}, \api{QBrush}, \api{QFont}, \api{QImage}, and \api{QPixmap}. The same painter concepts work for custom widgets, raster images, SVG generation, and PDF output.

\section{Paths}

\api{QPainterPath} supports path construction, element inspection, bounds queries, hit testing, translation, simplification, and path composition.

\begin{lstlisting}[style=crystal]
path = Qt6::QPainterPath.new
path.move_to(20, 20)
path.cubic_to(90, 0, 140, 120, 220, 40)
path.line_to(220, 160)
path.close_subpath

if path.contains(Qt6::PointF.new(80, 64))
  path.translate(12, 0)
end
\end{lstlisting}

\section{Painter State}

Use pens for strokes, brushes for fills, transforms for coordinate-space changes, and clipping to constrain drawing. When painting to image, SVG, or PDF devices, call the explicit painter shutdown helper before reading or saving the target.
